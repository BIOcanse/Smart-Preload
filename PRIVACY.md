# Smart Preload Privacy Policy

Last updated: June 23, 2026

Smart Preload is a browser extension that improves perceived page loading speed by learning local browsing patterns and preloading likely next pages.

## Summary

Smart Preload processes browsing-related data locally in your browser profile so it can rank links and manage preload candidates. The developer does not operate a server for collecting this data, does not sell it, and does not share it with advertisers or data brokers.

Some optional features may send limited data to services you choose, such as an AI provider endpoint you configure or GitHub when checking for native app releases.

## Data processed locally

Smart Preload may process and store the following data locally on your device:

- Page URLs, hosts, titles, and navigation transitions.
- Link candidates found on pages, including link URLs and nearby text signals used for ranking.
- Tab and window state needed to manage preloaded pages.
- Interaction signals such as link hover, context-menu preload intent, foreground tab activity, and recent active time.
- Bookmark titles and URLs when bookmark-based preload features are enabled.
- Extension settings, preload limits, safety settings, local history statistics, and diagnostic logs when diagnostics are enabled.

This data is used to provide the extension's core features: prediction, ranking, preload scheduling, safety filtering, local history deletion, and local diagnostics.

## Data sent outside your device

Smart Preload does not send browsing history or preload history to the developer.

Data may leave your device only in these cases:

- If you enable an external AI prediction provider and enter an API key or endpoint, the extension may send the selected page/link context needed for keyword or relevance scoring to that provider. The provider's own privacy policy applies to that request.
- If you use a local AI endpoint such as LM Studio, requests are sent to the endpoint you configured.
- If you check for or download native app updates, the extension or native app may contact GitHub release pages or GitHub-hosted files.
- Normal browser networking still occurs when you actually visit a page or when a preload feature loads a page in the browser.

## Native app

The optional Windows native app helps keep real preload windows hidden and supports local system integration. It communicates with the extension through Chrome/Edge native messaging on the same device. The native app does not send browsing history to the developer.

## Storage and deletion

Smart Preload stores its data in local browser extension storage and related local files used by the extension/native app. You can delete selected local history ranges from the extension settings page. You can also remove stored extension data by uninstalling the extension or clearing the browser profile's extension data.

## Permissions

Smart Preload requests browser permissions only to support its features, including reading tabs and navigation events, storing local settings and history, using bookmarks for optional bookmark preloading, communicating with the optional native app, and running scheduled maintenance checks.

## No sale of data

The developer does not sell, rent, or share user data with advertisers, analytics brokers, or other third parties for tracking or profiling.

## Children's privacy

Smart Preload is not directed to children and does not knowingly collect personal information from children.

## Changes

This policy may be updated when Smart Preload changes its data handling behavior. The latest version will be published in this repository.

## Contact

For privacy questions, contact: biocanse@gmail.com

# 智能预加载隐私政策

最后更新：2026 年 6 月 23 日

智能预加载是一个浏览器扩展，通过在本地学习浏览行为并预加载可能打开的页面，降低用户感知到的等待时间。

## 摘要

智能预加载会在你的浏览器配置文件中本地处理与浏览相关的数据，用于链接排序和预加载候选管理。开发者没有用于收集这些数据的服务器，不会出售这些数据，也不会与广告商或数据经纪商共享这些数据。

部分可选功能可能会把有限数据发送到你自己选择的服务，例如你配置的 AI 服务端点，或检查本地 app 更新时访问 GitHub。

## 本地处理的数据

智能预加载可能会在你的设备上本地处理和存储以下数据：

- 页面 URL、站点、标题和跳转关系。
- 页面中的链接候选，包括链接 URL 和用于排序的附近文本信号。
- 管理预加载页面所需的标签页和窗口状态。
- 链接悬停、右键菜单预加载意图、前台标签页活动和近期活跃时间等交互信号。
- 启用书签预加载功能时的书签标题和 URL。
- 扩展设置、预加载上限、安全设置、本地历史统计，以及开启诊断时的诊断日志。

这些数据用于提供扩展的核心功能：预测、排序、预加载调度、安全过滤、本地历史删除和本地诊断。

## 会离开设备的数据

智能预加载不会把浏览历史或预加载历史发送给开发者。

只有以下情况数据可能离开你的设备：

- 如果你启用外部 AI 预测服务并填写 API key 或服务端点，扩展可能会把关键词或相关性评分所需的页面/链接上下文发送给该服务。该服务自己的隐私政策适用于这些请求。
- 如果你使用 LM Studio 等本地 AI 端点，请求会发送到你配置的端点。
- 如果你检查或下载本地 app 更新，扩展或本地 app 可能会访问 GitHub release 页面或 GitHub 托管文件。
- 当你实际访问页面，或预加载功能在浏览器中加载页面时，浏览器本身仍会进行正常网络请求。

## 本地 app

可选的 Windows 本地 app 用于保持真实预加载窗口隐藏，并支持本机系统集成。它通过 Chrome/Edge native messaging 在同一台设备上与扩展通信。本地 app 不会把浏览历史发送给开发者。

## 存储和删除

智能预加载的数据存储在浏览器扩展本地存储以及扩展/本地 app 使用的相关本地文件中。你可以在扩展设置页删除指定时间范围内的本地历史记录。你也可以通过卸载扩展或清理浏览器配置文件中的扩展数据来移除已存储数据。

## 权限

智能预加载只为支持功能而请求浏览器权限，包括读取标签页和导航事件、存储本地设置和历史、为可选书签预加载使用书签、与可选本地 app 通信，以及运行定期维护检查。

## 不出售数据

开发者不会把用户数据出售、出租或共享给广告商、分析数据经纪商或其他第三方用于追踪或画像。

## 儿童隐私

智能预加载并非面向儿童，也不会有意收集儿童个人信息。

## 政策更新

当智能预加载的数据处理方式发生变化时，本政策可能会更新。最新版本会发布在本仓库中。

## 联系方式

隐私相关问题请联系：biocanse@gmail.com
