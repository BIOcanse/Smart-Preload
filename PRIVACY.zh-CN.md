# 智能预加载隐私政策

最后更新：2026 年 6 月 23 日

本政策适用于智能预加载浏览器扩展，以及可选的 Windows 本地配套 app。

智能预加载使用智能预加载算法来降低用户感知到的加载等待时间。为实现这一点，它会在你的浏览器配置文件和本机设备上处理与浏览相关的信号。开发者没有用于收集浏览历史的服务器，不会出售用户数据，也不会与广告商、分析数据经纪商或数据经纪商共享用户数据。

## 本地处理的数据

智能预加载可能会在本地处理和存储以下数据：

- 页面 URL、站点、标题和导航跳转关系。
- 页面中的链接候选，包括链接 URL、链接文本，以及用于排序的附近文本。
- 管理已准备页面所需的标签页、窗口、预加载和预取状态。
- 链接悬停、右键菜单预加载意图、前台标签页活动、近期活跃时间，以及调度所需的媒体/活动状态。
- 启用书签预加载功能时的书签标题和 URL。
- 扩展设置、预加载上限、安全设置、本地历史统计、AI 服务设置，以及开启诊断时的诊断日志。

这些本地数据只用于提供智能预加载的功能：预测、排序、预加载调度、安全过滤、本地历史删除、诊断和可选的 AI 辅助评分。

## 会离开设备的数据

智能预加载不会把浏览历史或预加载历史发送给开发者。

只有以下情况下数据可能离开你的设备：

- 如果你启用外部 AI 服务并填写 API key 或端点，扩展可能会把关键词或相关性评分所需的页面/链接上下文发送给该服务。该服务自己的隐私政策适用于这些请求。
- 如果你使用 LM Studio 等本地 AI 端点，请求会发送到你配置的端点。
- 如果你检查或下载本地 app 更新，扩展或 Windows 本地配套 app 可能会访问 GitHub release 页面或 GitHub 托管文件。
- 当你实际访问页面，或预加载功能在浏览器中加载页面时，浏览器会发生正常网络请求。目标网站可能会像普通页面加载一样接收请求、Cookie 和会话信息。

## 可选 Windows 本地 app

可选的 Windows 本地配套 app 用于保持真实预加载窗口隐藏，并支持本机系统集成。它通过 Chrome/Edge native messaging 在同一台设备上与扩展通信。它可能会在本地处理浏览器窗口元数据和本机系统性能/活动信息，以管理预加载行为。它不会把浏览历史发送给开发者。

## 存储和删除

智能预加载的数据存储在浏览器扩展本地存储，以及扩展或本地 app 使用的相关本地文件中。你可以在扩展设置页删除指定时间范围内的本地历史记录。你也可以通过卸载扩展或清理浏览器配置文件中的扩展数据来移除已存储数据。

## 权限

智能预加载只为支持功能而请求浏览器权限，包括标签页和导航事件处理、本地存储、基于书签的预加载功能、与可选本地 app 的 native messaging 通信，以及定期维护检查。

## 不出售或用于广告

开发者不会把用户数据出售、出租或共享给广告商、分析数据经纪商或其他第三方用于追踪、画像或广告。

## Chrome 网上应用店 Limited Use

智能预加载使用从 Chrome 扩展 API 获得的信息时，仅限于提供和改进其单一目的：通过本地预测和预加载降低用户感知到的页面加载等待时间。智能预加载对这些信息的使用遵守 Chrome Web Store User Data Policy，包括其中的 Limited Use 要求。

## 儿童隐私

智能预加载并非面向儿童，也不会有意收集儿童个人信息。

## 政策更新

当智能预加载的数据处理方式发生变化时，本政策可能会更新。最新版本会发布在本仓库中。
